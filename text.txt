Good — that means the **embedding side is finally healthy** with `ollama:nomic-embed-text`. `nomic-embed-text` is an Ollama embedding-only model, and Ollama explicitly describes it as usable only for generating embeddings. ([Ollama][1])

Now do these steps.

## 1. Confirm the model dimension once

```bash
curl http://localhost:11434/api/embed \
  -H "Content-Type: application/json" \
  -d '{"model":"nomic-embed-text","input":"test"}' \
  | jq '.embeddings[0] | length'
```

Expected:

```text
768
```

Then:

```bash
gbrain providers test --touchpoint embedding

```

If those pass, move on.

## 2. Create or verify your `ai-brain` repo

Do **not** put this inside `~/gbrain`. Use a separate repo:

```bash
cd ~
```

Create the starter files:

```bash
cat > README.md <<'EOF'
# AI Brain

Personal/project knowledge base for GBrain.
EOF

cat > RESOLVER.md <<'EOF'
# Resolver

- inbox/ — temporary captures that need sorting
- projects/ — long-running projects
- repos/ — repo-specific architecture summaries and current state
- workflows/ — Claude Code, Codex, Pi, GStack, PR review, CI/CD patterns

- prompts/ — reusable prompts for agents
- concepts/ — explanations and mental models
- sources/ — article/research/source notes
- people/ — people notes
- companies/ — company/tool/vendor notes
- archive/ — old or superseded material

Before creating a new page, search for an existing page first.
Do not store secrets, private keys, tokens, passwords, or raw credentials.
EOF

cat > index.md <<'EOF'
# Index

Start here for the brain.

Important areas:
- projects/
- repos/
- workflows/
- decisions/
- prompts/
- concepts/
- sources/
EOF

cat > schema.md <<'EOF'
# Schema

Preferred page shape:

# Title

## Current truth

The current best understanding.

## Details

Supporting details.

## Timeline / evidence

- YYYY-MM-DD — Source or event.

## Links

Related pages.
EOF

for d in inbox projects repos workflows decisions prompts concepts sources people companies archive; do
  [ -f "$d/README.md" ] || echo "# $d" > "$d/README.md"
done
```

Commit it:

```bash
git add .
git commit -m "Initialize AI brain"
```

GBrain’s README describes the brain repo as the source-of-truth Markdown/git repo, while GBrain syncs it into PGLite/Postgres for retrieval. ([GitHub][2])

## 5. Connect Claude Code locally

Create a wrapper:

```bash
mkdir -p ~/bin

cat > ~/bin/gbrain-mcp <<'EOF'
#!/usr/bin/env bash
set -euo pipefail

cd "$HOME/ai-brain"

export PATH="$HOME/.bun/bin:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:$PATH"
export OLLAMA_HOST="${OLLAMA_HOST:-http://127.0.0.1:11434}"

exec gbrain serve
EOF

chmod +x ~/bin/gbrain-mcp
```

Test it:

```bash
~/bin/gbrain-mcp
```

It should hang silently. Press `Ctrl+C`.

Add to Claude Code:

```bash
claude mcp remove gbrain 2>/dev/null || true
claude mcp add --scope user gbrain -- "$HOME/bin/gbrain-mcp"
claude mcp list
```

GBrain’s MCP docs say local stdio via `gbrain serve` works with Claude Code, Cursor, Windsurf, and other stdio MCP clients, with no server/tunnel/token needed. ([GitHub][3])

## The short version

You are now past the hard part. Next steps are:

```bash
# 1. Make/import brain repo
gbrain import ~/ai-brain
gbrain embed --stale
gbrain query "What is this brain for?"

# 2. Add Claude MCP
claude mcp add --scope user gbrain -- "$HOME/bin/gbrain-mcp"

# 3. Add CLAUDE.md guidance to your repo
cd ~/GitHub/coupir
nano CLAUDE.md

# 4. Use it
claude
```

At this point, GBrain is useful as **repo memory**: architecture notes, prior decisions, prompts, workflow rules, bugs, and “don’t repeat this mistake” knowledge. Full code indexing is optional later; the stable win is making Claude Code search your durable project memory before it starts blindly reading files.
