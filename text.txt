Yep — replace the `.clamp(...)` call with a small `Duration` clamp helper.

`clamp()` is only on `num`, not `Duration`, which is why this fails. `Duration` can still be compared using `<` / `>` / `compareTo`, so this helper is the clean fix. Dart’s docs show `num.clamp(...)` exists on `num`, while `Duration` is a separate class with comparison support. ([Flutter API Docs][1])

Change this:

```dart
return switch (errorCount) {
  1 => Duration.zero,
  2 => const Duration(milliseconds: 250),
  3 => const Duration(milliseconds: 500),
  _ => const Duration(seconds: 1),
}.clamp(Duration.zero, const Duration(seconds: 2));
```

to this:

```dart
return _clampDuration(
  switch (errorCount) {
    1 => Duration.zero,
    2 => const Duration(milliseconds: 250),
    3 => const Duration(milliseconds: 500),
    _ => const Duration(seconds: 1),
  },
  Duration.zero,
  const Duration(seconds: 2),
);
```

Then add this helper above or below the class:

```dart
Duration _clampDuration(
  Duration value,
  Duration min,
  Duration max,
) {
  if (value < min) return min;
  if (value > max) return max;
  return value;
}
```

So your file should look like:

```dart
/// Step-function backoff: 1 error → 0ms, 2 → 250ms, 3 → 500ms, 4+ → 1s.
class StepBackoffStrategy implements BackoffStrategy {
  const StepBackoffStrategy();

  @override
  Duration computeDelay(int errorCount) {
    return _clampDuration(
      switch (errorCount) {
        1 => Duration.zero,
        2 => const Duration(milliseconds: 250),
        3 => const Duration(milliseconds: 500),
        _ => const Duration(seconds: 1),
      },
      Duration.zero,
      const Duration(seconds: 2),
    );
  }
}

Duration _clampDuration(
  Duration value,
  Duration min,
  Duration max,
) {
  if (value < min) return min;
  if (value > max) return max;
  return value;
}
```
